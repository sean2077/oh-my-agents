# Changelog

> 发布流程：用户决定发布 → 更新本文件（内容面向 GitHub Release 页面读者）→ commit + tag `vX.Y.Z` → push 触发 CI → CI 用 `scripts/extract-changelog.sh` 截取与 tag 同名的版本节作为 Release 正文。
> 版本节标题格式：`## vX.Y.Z - YYYY-MM-DD`（CI 按 tag 前缀精确匹配；tag 无对应版本节 → 发布失败，fail-closed）。
